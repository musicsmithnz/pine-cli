    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#roadmap" aria-hidden="true" class="anchor" id="user-content-roadmap"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Roadmap</h1>
<p>An overview of what the core Polymer team is looking to work on over the next few months.</p>
<p>As a caveat - this is a living doc, and will evolve as priorities grow and shift. The Polymer project will always be adapting to new use-cases and evolutions in the platform - this roadmap is more of a "North Star" of what we're looking to work on than a strict timeline.</p>
<p>Please feel free to file issues on this repository if you have questions, concerns, or suggestions.</p>
<p>Last Updated: 1/18/2018 by arthurevans</p>
<h3><a href="#polymer-30-preview" aria-hidden="true" class="anchor" id="user-content-polymer-30-preview"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Polymer 3.0 (preview)</h3>
<p>We announced an early sneak preview of Polymer 3.0 at <a href="https://www.youtube.com/watch?v=TDpiyrcOO30&amp;list=PLNYkxOF6rcIDP0PqVaJxqNWwIgvoEPzJi" rel="nofollow">Polymer Summit 2017</a>. We then posted a two-part series on <a href="https://www.polymer-project.org/blog/2017-08-22-npm-modules.html" rel="nofollow">what Polymer 3.0 entails</a> and <a href="https://www.polymer-project.org/blog/2017-08-23-hands-on-30-preview.html" rel="nofollow">how to get started</a> with it.</p>
<p>In summary: We're migrating to npm from Bower; migrating to ES Modules from HTML Imports; and not planning to change the API from Polymer 2.0. Because of that, the changes will be minimal and mechanical, so we built the Polymer Modulizer to automate the upgrade.</p>
<p>We're planning to release Polymer 3.0 by the end of Q1, 2018. Until the release, we're planning on frequent updates
to the preview code on npm under the <code>@next</code> tag.</p>
<p>Most 3.0-related work is taking place in the <a href="https://github.com/Polymer/polymer-modulizer">Polymer Modulizer</a> repo.</p>
<p>Most elements are now being automatically converted for 3.0, but in many cases tests aren't running at all or some tests are broken. For details, see the <a href="https://github.com/Polymer/polymer-modulizer/blob/master/docs/polymer-3-element-status.md">Polymer 3.0 element status page</a>.</p>
<p>The following items are on the roadmap for Polymer 3.0:</p>
<ul>
<li>Better support for incremental conversion in Modulizer. Instead of requiring you to convert an entire set of packages
in one batch, Modulizer will output a manifest of the conversion for each package. The manifest can be published to npm
along with the converted package. When converting, Modulizer will read the published manifests of dependencies
and skip conversion for dependencies that have already been converted and published.</li>
<li>JavaScript Module bundling. Polymer Bundler will produce either HTML imports based bundles or JS modules based bundles.</li>
<li>Bare-specifier support: Analyzer will resolve them, <code>polymer-build</code> and <code>polyserve</code> will rewrite them.</li>
<li>Support for dynamic <code>import()</code>: <code>polymer-build</code> and <code>polyserve</code> can rewrite these when compiling out modules.</li>
<li>Support for <code>import.meta.url</code>: Polymer Bundler, <code>polymer-build</code> and <code>polyserve</code> will rewrite these references for
browsers that don't support this property.</li>
<li>Updated documentation for Polymer 3.0.</li>
<li>Updated versions on Polymer Starter Kit and other templates for Polymer 3.0.</li>
</ul>
<h3><a href="#polymer-20" aria-hidden="true" class="anchor" id="user-content-polymer-20"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Polymer 2.0</h3>
<p>Polymer 2.0 was announced at <a href="https://www.youtube.com/watch?v=cuoZenpQveQ&amp;t=981s" rel="nofollow">Google I/O 2017</a>. Along with our main keynote, we had 2 talks on <a href="https://www.youtube.com/watch?v=assSM3rlvZ8&amp;t=18s" rel="nofollow">building elements</a> with Polymer 2.0 and our <a href="https://www.youtube.com/watch?v=tKvNeNGmOtU&amp;t=96s" rel="nofollow">new tools</a>.</p>
<p><em>Goals</em>:</p>
<ul>
<li>Make the Polymer library compatible with latest web component specs and shipping implementations.
 - <g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Update styling system to be compatible with custom properties.
<ul>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Update shady DOM to be compatible with shadow dom v1.</li>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Update custom elements treatment to be compatible with custom elements v1.</li>
</ul>
</li>
<li>Provide a smooth migration from Polymer 1.x.
<ul>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Provide backwards-compatibility layer that re-creates the <code>Polymer({})</code> factory from 1.x.</li>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Design a "hybrid mode" so that elements built in a certain way can run under 1.x or 2.0.</li>
</ul>
</li>
<li>Eliminate leaky abstractions.
<ul>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Remove <code>Polymer.dom</code> API.</li>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Remove the <code>set/notifyPath</code> API's.</li>
</ul>
</li>
<li>Support ES6 class-based syntax as primary way to build new elements.
<ul>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> <code>class MyElement extends Polymer.Element</code> !!</li>
</ul>
</li>
<li>Improve factoring for more flexible usage.
<ul>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Break out "Shady DOM" shim from Polymer.</li>
<li><g-emoji class="g-emoji" alias="heavy_check_mark" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/2714.png" ios-version="6.0">✔️</g-emoji> Break out CSS property shim from Polymer.</li>
</ul>
</li>
</ul>
<h3><a href="#whats-next" aria-hidden="true" class="anchor" id="user-content-whats-next"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>What's next?</h3>
<p>We're currently researching a variety of things related to building Web Components and Progressive Web Apps, given that so much has changed in the web platform since we released our first production-quality version of the Polymer library. Stay tuned for blog posts and updates on our progress there, and check out <a href="https://developer.chrome.com/devsummit/" rel="nofollow"><em>Chrome Developer Summit</em></a> on October 23rd and 24th to hear about PWAs with Taylor Savage, and more on lit-html with Justin Fagnani-Bell.</p>
<h3><a href="#community-and-ecosystem" aria-hidden="true" class="anchor" id="user-content-community-and-ecosystem"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Community and Ecosystem</h3>
<p>The Polymer community has grown incredibly quickly and there are a number of individuals making extremely meaningful contributions to both the library and Polymer elements. We want to do more to highlight community work and generally better serve the Polymer and web components ecosystem. This means even more transparency around the core Polymer team's roadmap, more regular announcement and updates, and promoting element ownership beyond just the core Polymer team.</p>
<p><em>Goals</em>:</p>
<ul>
<li>Accelerate Polymer + Community engagement.
<ul>
<li>Tighten the core Polymer team and community feedback loop - better evolve Polymer as the community evolves.</li>
<li>Publish all elements to npm and bower.</li>
<li>Regularly publish project and roadmap updates.</li>
<li>Work with community members to help drive elements forward.</li>
<li>Use <a href="https://www.youtube.com/playlist?list=PLOU2XLYxmsII5c3Mgw6fNYCzaWrsM3sMN" rel="nofollow">Polycasts</a> to highlight awesome elements being built.</li>
</ul>
</li>
</ul>
